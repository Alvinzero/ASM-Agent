### ASM Specification Requirements *(mandatory when ASM is in scope)*

ASM features MUST cite the JSON spec at `src/shared/spec/hk64s8x.v0.1.json`
and state which instruction, register, bit field, memory, vector, or syntax
surface is affected. The specification MUST require model-facing ASM generation
to send the user's requirement together with `SPEC_DRIVEN_ASM_CONTEXT`, and MUST
require the external model to directly return the deliverable ASM code block.
The specification MUST require numeric register, `JMP`, and `CALL` addresses to
use ASMC `H` suffix form such as `38H`, `46H`, and `20H`, and MUST reject bare
decimal or `0x` addresses as invalid final ASM.
The specification MUST forbid JSON-absent pseudo instructions such as `ORG`,
`END`, `EQU`, `DB`, `DS`, `DW`, `SECTION`, and `INCLUDE`. It MUST state that `R`
in instruction syntax is an operand placeholder rather than a real register, so
temporary counters use valid RAM `H`-suffix addresses such as `80H`, not invented
`R0`/`R1`/`R2` registers.
The specification MUST require software delay counters to use write-back skip
instructions such as `DECSZR`/`INCSZR`, not `DECSZ`/`INCSZ`, because
non-write-back skip instructions can keep LED code stuck in one visible state.
The specification MUST require software delay loops to be estimated against the
requested clock. For 16MHz and 500ms LED blinking, `#0AH/#FFH/#FFH` is the
expected three-level `DECSZR` scale, while `#7AH/#FFH/#FFH` is far too long.
The specification MUST forbid hidden main-path local rewrites through
`createPlan`, `generateValidatedAsm`, templates, or one-click normalization.
Generated ASM MUST pass local quality validation before display or file output;
validation failures MUST reject the output instead of saving it as final ASM.
