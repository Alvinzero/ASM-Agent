## ASM Agent 项目深度架构通读

通读了整个 ASM Agent 项目的源码，建立完整架构认知。

### 关键架构事实
- **双运行时**: Electron 桌面端 + Vite dev-server 浏览器端。
  - Electron: 主进程 IPC handlers (`agent:*`/`auth:*`/`file:*`/`project:*`)，preload 注入 `window.asmAgent`。
  - 浏览器: `vite.config.ts` 的 `localModelProxyPlugin` 提供 `/api/*` 中间件（auth/complete-chat/complete-chat-stream/session-file）。
  - 渲染端 `getAsmAgent()` 优先用 `window.asmAgent`，否则降级到 `BrowserModelProxy` + 浏览器 fallback agent。
- **规范事实源**: `src/shared/spec/hk64s8x.v0.1.json` (chipId=HK64S8x, 65 条指令, 96 个寄存器)。由 `scripts/compileSpec.ts` 从 xlsx 编译，含三列交叉一致性自检 (`assertInstructionConsistency`)。
- **ASM 生成有两条路径**:
  1. 本地规则引擎路径: `LocalRuleAgent.createPlan` (关键词匹配 Timer0/GPIO/Interrupt/WDT) → `SingleAsmFileGenerator` (模板渲染) → 质检。
  2. 外部模型直出路径: `appendSpecPromptAttachment` 注入 `SPEC_DRIVEN_ASM_CONTEXT` → 模型返回 ```` ```asm ```` 代码块 → `buildModelAsmGeneratedFile` 仅抽取+质检(含 PA 彩灯行为/延时估算)，不本地改写。质检失败可重试 1 次。
- **质检闸门**: `parseAsm` → `validateAsm` (指令/寄存器/标签/操作数/H后缀/范围) → `InstructionEncoder` mask/value 复检。`ProjectGenerator` 把所有 .asm 合并后做跨文件标签校验。
- **核心编排**: `src/renderer/state/useAgentSession.ts` (~1900 行) 是大脑。意图分类 (ordinaryChat/asmHelp/asmGeneration/unclearGeneration) → 对应走模型流式/本地规则。Codex 风格 trace 时间线 (narration/commands/action/edit/reasoning)。
- **会话持久化**: SQLite (auth, `node:sqlite` + scrypt) + LocalStorage (会话快照/模型配置/面板宽度)。多会话并发任务由 `SessionTaskManager` 单例管理，切换会话不中断后台任务。
- **AGENTS.md 强约束**: H 后缀地址、禁伪指令、PA 彩灯掩码、DECSZR 回写、延时估算、模型直出不本地重写、质检必过。

## OLED I2C 点亮 main.asm 手写交付 (v1→v2→v3→v4→v5)
用户需求: HK64S8x + PB6(SCL)/PB7(SDA) 软件 I2C + SSD1306 + 全屏点亮。
- v1: RL 位移, 灯不亮。
- v2: 硬编码+17命令, MTP 容量溢出"数据长度错误"。
- v3: 修位逻辑反转+精简115行, 仍报 MTP 错误。
- v4: PB 极简配置+延时加长+总线复位134行, 仍报 MTP 错误。
- v5 (当前): 81条指令通过质检。关键发现+修复:
  1. **根因6 — 位操作指令触发编译器/烧录器 MTP 错误**: 对比能工作的彩灯版本(只用 MOV/CALL/JMP/CLRWDT/RET)和 v4(加了 BSET/BCLR/BTSZ/DECSZR)，v5 彻底移除 BSET/BCLR/BTSZ/RL/RR/RLC/RRC，只用 MOV/CALL/JMP/CLRWDT/RET/AND/OR/ADD/DECSZR。
  2. **I2C 端口操作改为整字节 MOV**: 不再用 BSET/BCLR 单独控位，改用 MOV 4 种组合值(0x00=SCL0SDA0, 0x40=SCL1SDA0, 0x80=SCL0SDA1, 0xC0=SCL1SDA1)。
  3. **位测试改用 AND**: `AND A,#080H` 后 A 本身就是 0x80 或 0x00，直接当端口值写入。
  4. **字节左移改用 ADD**: `ADD A,82H` (= A+A = A<<1) 代替 RL 82H，避免 RL 语义歧义。
  5. **注释纯 ASCII**: 移除所有 Unicode 箭头符号(↓↑→)，防止编译器编码问题。
  6. **格式贴近生成器**: 逗号后无空格 `MOV A,#0C0H`，跟 SingleAsmFileGenerator 输出一致。
- v1-v5 bug 链: RL位移(v1)→位逻辑反转(v2)→MTP溢出(v2)→PB开漏弱上拉(v3)→位操作指令触发MTP错误(v5)。
